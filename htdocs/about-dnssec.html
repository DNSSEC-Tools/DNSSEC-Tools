Domain Name System Security (DNSSEC)



The Domain Name System (DNS) translates names to some information about that name, typically an IP address. Most users trust the Internet’s system of domain names and expect to be directed reliably to the website they’ve entered in a browser. Unfortunately, that’s not always the case. Attackers can disrupt the DNS by forging network packets or gaining illicit access to servers on the network to corrupt or destroy information.

DNS security (DNSSEC) works by introducing digital signatures throughout the DNS infrastructure. It establishes that the binding between a domain name and its resource records, including its IP addresses, has not been compromised. DNSSEC  proves the origin authenticity of the DNS data: that is, it proves that the DNS data originated from an authorized entity and wasn't modified in transit. It is the one true defense against cache poisoning attacks on the recursive name server, and since it provides end-to-end DNS data integrity, it also protects the user against compromised secondary name servers that may be managed by different organizations half way across the world.

DNS threats are invisible to the end user, therefore validation must always be enabled to protect the user against spoofed responses that the user does not necessarily see. Ubiquitous DNSSEC validation also enables the creation of a number of new applications that can rely on a secure DNS infrastructure as its base.
