Setting Up the Owl Monitoring System

The Owl Monitoring System monitors basic network functionality, using timed Domain Name System (DNS) queries. The system consists of a manager host and a set of sensor hosts. The sensors perform periodic DNS queries and report to the Owl manager the time taken for each query. Over time, this shows the responsiveness of the DNS infrastructure.

The Owl Monitoring System was designed such that the Owl sensor hosts need not be under the same administrative control as the Owl manager host. In fact, each sensor in a set of Owl sensors may be under administrative control of different entities and still report to a single Owl manager. Also, an Owl sensor may provide its data to more than one Owl manager. All contact between sensors and manager is initiated and performed by the sensor; the Owl manager does not attempt to pull data from any of its sensors.

This document provides an operational overview of the Owl systems, installation instructions for the Owl and supporting third-party software, and configuration instructions for the Owl software.

How To Use This Manual

The readers of this manual fit into one of three roles: administrator of the Owl manager and Owl sensors, administrator of the sensors only, and administrator of the manager only. This brief guide will show the manual sections that must be read by each role. The top-level sections should be read by all roles -- with the exception of section 5, which need not be read the sensor-only administrators.

Administrators in this role must read the entire installation manual.
